✅ Please follow the below steps to configure the ansible master/controller node

1) Create a dedicated user for ansible, I am using ansibleuser and this user must have a sudo privileges.
2) SSH connection to be there between controller node and targeted nodes
3) Python must be there on all nodes.
4) If using hostname then name resolution via DNS or /etc/hosts file.
5) Create inventory file on controller node and ansible.cfg file for default configurations and parameters.

** A quick setup, I have created a custom file for ansibleuser for sudo privilege and ansible.cfg file, present in github repo, clone the same repo and follow below steps:
1) git clone https://github.com/path4cloud2024/ansible-ops.git
2) creating a new user on ansible control node:
    # useradd ansibleuser
    # su - ansibleuser (switching to a new user)
    # ssh-keygen (creating a SSH KEY)
    # copy the "ansibleuser" file from cloned repo to /etc/sudoers.d/
    Note: If you have another user then replace the "ansibleuser" with your username
    # sed -i "s|ansibleuser|<new_user>|g" ansibleuser
3) copy and rename the anisble.cfg.bkp file to ansibleuser home directory as ansible.cfg (hidden file)
    # cp ansible.cfg.bkp /home/ansibleuser/.ansible.cfg
4) copy the inventory file to user home directory and update as per your setup.
    # cp inventory /home/ansibleuser/inventory
5) Now, run the playbook ansibleuser.yaml from ansibleuser home directory. This playbook prompt for user,provide the user as "ansibleuser" (OR, if you have created another user then provide the same username)
    Note: If you have another user which has already present in target node and has sudo privilege then at line num 20, replace the "path4cloud" user to an user which is already present user.
    # ansible-playbook ansible-ops/ansibleuser.yaml -k -K (will ask the password twice for user mentioned in line 20)
6) check the new user is working with privilege access or try to ping as adhoc command:
    # ansible docker -m ping 
    # ansible docker -a "mkdir /root/testdir" (here docker is my managed host and already mentioned in inventory file, you can change it to your name)
