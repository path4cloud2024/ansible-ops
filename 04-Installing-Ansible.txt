🔹 Control Node/ Master Node

    - We can setup a control node (install Ansible) on RedHat family Linux (RHEL/ CentOS)
    - It can be installed on debian based operating system - Ubuntu
    - With Python pip also, we can configure the Ansible Master node. (but not supported from RH, only community based support)
    - We CANNOT INSTALL ANSIBLE on WINDOWS MACHINE.
    - Ansible can be installed on MAC machine as well.

🔹 Control Node components:
- Package to install "ansible-core" or "ansible"
- ansible.cfg file for default configuration of master node
- An inventory file where we define the target nodes to manage (defualt inventory file /etc/ansible/hosts)

🔹 Various Format of Ansible Package
- Ansible Core: It is just the base of ansible, you need to install other components via collections.
- Community Ansible: It is top of Ansible Core with very few collections already embedded with Ansible..
- RedHat Ansible: Installed via RedHat official repositories. Also, comes with Redhat Subscription and  we can have these 3 additional features:
    * RHEL system Roles
    * RedHat Insight Remediation 
    * OpenSCAP Security Profile Configuration
- Ansible Automation Platform Subscription (comes with Ansible Controller)

✅ Install Ansible-Core 
    ** If server image is used from console.redhat.com and registered to CDN network then we do have ansible-core installed at part of RHEL subscription with system (but we have limited support)

    ** Server should be registered to some upstream network, In this LAB, our server is registered to console.redhat.com and using "Red Hat Developer Subscription for Individuals" Subscription

    (mostly it is attached to this Developer pools if not then run:)
    # subscription-manager attach --auto
    OR 
    # subscription-manager list --available
    # subscription-manager attach --pool=<POOL_ID>

    check the repositories:
    # dnf repolist all
    (if ansible repo is not enabled then we can enable as:)
    # subscription-manager repos --enable=ansible-automation-platform-2.6-for-rhel-9-x86_64-rpms
    (here, we are enabling 2.6 version, you can choose as per your requirement)
    # yum install ansible-core -y

    To install the ansible navigator
    # yum install ansible-navigator -y

    From Community (EPEL) repo
    Configure the EPEL Repo (EPEL 9 Everything x86_64)
    # cd /etc/yum.repos.d/
    # vi epel.repo
        [EPEL-9-Everything-x86_64]
        name=EPEL 9 Everything x86_64
        baseurl=https://dl.fedoraproject.org/pub/epel/9/Everything/x86_64/
        gpgcheck=1
        repo_gpgcheck=0
        gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-2f0b8a71-c837-45b8-b7d0-2eb86cc3d0bd-0
        module_hotfixes=0
    
    Save this file, and run 
    # yum repolist
    OR
    # dnf install epel-release (to configure automatically)
    # yum install ansible -y (CLI version)
    
    Once installed, check version
    # ansible --version

🔹 How Community Contribution Works:
    You can create your own module and maintain it on github or any other portal and publish that module. Now, ask comunnity (by filling forms) to test the modules in their environment and give some feedback, we can also ask RedHat to check our modules and if Redhat likes that then they will test it Redhat environment and then if it gets approved then it will get involved/shipped or made available on ansible automation hub portal for future version of ansible.
    If we are using ansible-core then we can fetch it from community and install in our environment.
